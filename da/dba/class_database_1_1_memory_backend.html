Store person's fingerprint in db.

  • fp Person's fingerprint
    Returns
    fingerprint id assigned by db
    If fp has valid id but empty fingerprint, it will be removed from db.
    If fp has invalid id but valid fingerprint data, new entry will be added to db.
    If fp has both entries valid, fingerpritn data for given id will be updated.

Person::Id Database::MemoryBackend::store ( const PersonName &  pn)
overrideprivatevirtual

Store or update person.

  • pn Details about person name to be stored.
    Returns
    id of created or updated person.
    If pn has valid id then person name will be updated in database
    If id is not valid and pn holds name already existing in database
    then method will return id of that person.
    If id is not valid and name was not found in database then
    new person will be added to database and its id will be returned.
